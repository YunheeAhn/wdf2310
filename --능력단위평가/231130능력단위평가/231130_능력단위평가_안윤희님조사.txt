가상 클라이언트의 요구사항
-> 사용자들이 사용하기에 불편한 온라인 예약 시스템과
    명확하지 않은 다양한 의료 정보 접근성 개선



클라이언트가 생각하는 문제의 원인 파악
-> 다양한 디바이스에 최적화 되지 않은 옛날 스타일의 웹사이트
-> 느린 웹사이트 첫 로딩
-> 메뉴 및 서비스에 대한 직관적인 의료정보 탐색 경로 도입
-> 시각적 디자인 개선
-> 일관적이지 않은 폰트 사이즈, 브랜드 컬러
-> 환자들을 위한 직관적이지 않고 편리해보이지 않는 예약 시스템
-> 웹사이트상의 의료진, 진료과목 정보 미비



해결방법
-> 다양한 디바이스에 대응할 수 있도록 일관된 반응형 웹사이트 제작
-> 빠른 로딩을 위한 웹사이트 최적화 하여 접근성 개선
-> 사용자 친화적인 의료정보, 약물정보, 진료 비용 확인 등 다양한 기능 개선
-> 브랜드 메인 컬러, 폰트 사이즈를 각 중요도에 맞게 개선
-> 사용자들이 쉽게 이해 할 수 있도록 직관적이고 단순한 형태를 지향하는 예약 시스템 UI 개선
-> 메인 배너에 핵심 의료진 홍보 이미지 개시 및 클릭시 의료진 정보 또는 진료과목 정보 제공 페이지로 이동
-> 병원의 이미지와 브랜딩 강화
